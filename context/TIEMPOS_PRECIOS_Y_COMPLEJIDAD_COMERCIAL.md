# Tiempos, Precios y Complejidad Comercial

**ID:** ARCH-20260423-19  
**Proyecto:** Agencia de Marketing AI-First  
**Fecha:** 2026-04-23  
**Estado:** Guia operativa inicial para cierre de sprint

## 1. Proposito

Este documento responde tres preguntas practicas del modelo comercial:

1. cuanto deberia durar cada etapa o producto,
2. que rango de precio sugerido puede tener cada uno,
3. como cambia el alcance cuando el cliente tiene una sola oferta, varias ofertas o un catalogo muy amplio.

Tambien responde una cuarta pregunta implicita:

4. como distinguir entre tiempo de implementacion y permanencia sugerida dentro de cada etapa.

No fija tarifas finales. Fija una primera logica operativa para vender sin improvisar.

## 2. Regla Base

Los paquetes no deben crecer en alcance al mismo ritmo que crece la cantidad de productos del cliente.

Si no se controla esa relacion, el margen desaparece.

La regla correcta es:

**los paquetes se diseñan alrededor de una unidad comercial focal, no alrededor de todo el universo del cliente.**

Esa unidad comercial focal puede ser:

1. un negocio nuevo con una sola oferta principal,
2. una linea prioritaria de servicio,
3. un producto ancla,
4. una categoria principal,
5. una campaña puntual de una familia de productos.

## 3. Tiempos Completos Sugeridos por Producto

Para evitar confusion, cada producto debe leerse en dos tiempos distintos:

1. tiempo de implementacion o setup,
2. permanencia sugerida dentro de la etapa antes de reevaluar, expandir o cambiar de nivel.

La implementacion responde: cuanto tarda en montarse o entregarse.

La permanencia responde: cuanto tiempo tiene sentido sostener ese nivel para generar valor y decidir el siguiente paso.

| Producto | Implementacion sugerida | Permanencia sugerida en la etapa | Naturaleza |
| :-- | :-- | :-- | :-- |
| Lanzamiento Comercial Inicial | 2 a 4 semanas | 1 a 2 meses | Proyecto de arranque |
| Presencia que Vende | 2 a 5 semanas | 1 a 3 meses | Proyecto de orden y consolidacion |
| Funnel de Captacion | 2 a 4 semanas de setup y activacion | 2 a 3 meses | Activacion y aprendizaje de growth |
| Motor de Contenido Funcional | 1 a 2 semanas de setup editorial y operativo | 2 a 6 meses o mas | Servicio recurrente o por ciclos |

## 4. Lectura de Tiempos por Etapa

### Lanzamiento Comercial Inicial

Implementacion:

Menos de 2 semanas suele implicar prisa excesiva y baja calidad.

Mas de 4 semanas empieza a parecer proyecto abierto y puede romper margen.

Permanencia:

Normalmente no deberia quedarse mas de 2 meses en este nivel sin tomar una decision. Si pasa ese tiempo y aun no puede avanzar, probablemente la oferta o la operacion del cliente siguen inmaduras.

### Presencia que Vende

Implementacion:

Menos de 2 semanas solo funciona si el cliente ya tiene bastante material ordenado.

Mas de 5 semanas normalmente significa que ya se esta convirtiendo en rediseño amplio.

Permanencia:

Entre 1 y 3 meses suele ser razonable para consolidar cambios visibles y decidir si ya puede pasar a captacion o si aun necesita terminar de ordenar su frente comercial.

### Funnel de Captacion

Implementacion:

Menos de 2 semanas suele ser poco realista para configurar bien activos, anuncios y medicion.

Mas de 4 semanas de setup sin salir a mercado indica friccion o sobrealcance.

Permanencia:

Menos de 2 meses rara vez da suficiente lectura. Entre 2 y 3 meses suele ser la ventana minima seria para decidir si conviene optimizar, escalar o redirigir.

### Motor de Contenido Funcional

Implementacion:

El setup editorial y operativo puede montarse en 1 a 2 semanas si el cliente tiene insumos y foco.

Permanencia:

Menos de 2 meses casi no deja ver continuidad real. Entre 2 y 6 meses es una ventana razonable para observar si la cadencia y el sistema generan valor sostenible. Despues de eso ya debe tratarse como relacion recurrente formal o retainer.

## 5. Precios Sugeridos Iniciales

Estos rangos estan pensados para clientes pequeños y medianos en una etapa inicial de agencia, con foco en claridad, rapidez y control de alcance.

| Producto | Precio sugerido MXN | Tipo de cobro |
| :-- | :-- | :-- |
| Lanzamiento Comercial Inicial | 18,000 a 35,000 | Pago por proyecto |
| Presencia que Vende | 15,000 a 30,000 | Pago por proyecto |
| Funnel de Captacion | 20,000 a 40,000 de setup inicial | Pago por proyecto o fase inicial |
| Motor de Contenido Funcional | 12,000 a 28,000 por ciclo mensual | Recurrente o por ciclo |

### Nota importante sobre precios

Estos precios no incluyen por defecto:

1. inversion en medios,
2. gastos de terceros,
3. compra de dominios, hosting o herramientas,
4. produccion fisica,
5. extensiones fuera del alcance base.

Son rangos de referencia para el servicio de la agencia, no para el costo total integral del ecosistema del cliente.

## 6. Lectura de Precio por Valor

El precio sugerido no debe verse como una cifra arbitraria. Debe justificarse con cinco variables:

1. claridad del problema que resuelve,
2. impacto comercial potencial,
3. velocidad de implementacion,
4. nivel de supervision humana requerido,
5. riesgo de desborde operativo.

### Justificacion general del pricing

La agencia no esta cobrando horas sueltas ni piezas aisladas. Esta cobrando:

1. criterio,
2. sistema,
3. velocidad de ejecucion,
4. reduccion de incertidumbre,
5. una ruta comercial mas clara para el cliente.

### Lanzamiento Comercial Inicial

Se vende por urgencia y claridad.

El precio lo sostiene la velocidad para dejar una base comercial util.

Se justifica porque:

1. reduce el tiempo de salida al mercado,
2. concentra multiples activos base en un solo esfuerzo controlado,
3. evita que el cliente arranque de forma improvisada,
4. abre la puerta a nuevos productos de mayor valor.

### Presencia que Vende

Suele ser noble comercialmente porque el cliente ya entiende que tiene algo desordenado y visible.

El precio lo sostiene la mejora directa en presentacion y conversion.

Se justifica porque:

1. corrige fricciones visibles del frente comercial,
2. mejora confianza y claridad sin rehacer todo el negocio,
3. puede impactar conversion con una intervencion relativamente acotada,
4. estandariza muy bien si se limita a activos clave.

### Funnel de Captacion

Tiene mayor cercania al ingreso.

El precio lo sostiene la promesa de generar aprendizaje real y demanda medible, no solo activar anuncios.

Se justifica porque:

1. acerca la agencia a resultado economico directo,
2. exige mayor responsabilidad en activacion y lectura,
3. requiere mejor criterio de diagnostico y readiness,
4. puede abrir relaciones de mayor ticket o recurrencia.

### Motor de Contenido Funcional

Debe cuidar muchisimo la relacion entre volumen y margen.

El precio lo sostiene la consistencia, la utilidad comercial y la continuidad, no la cantidad sin control.

Se justifica porque:

1. crea continuidad donde muchos clientes fallan,
2. traduce ideas del negocio en activos reutilizables,
3. sostiene presencia y confianza sin depender de produccion artesanal lenta,
4. puede convertirse en relacion recurrente si esta bien limitado.

## 6.1 Justificacion del Precio por Rango

La lectura correcta del rango es esta:

### Tramo bajo del rango

Aplica cuando:

1. hay una sola oferta,
2. los activos son pocos,
3. el cliente aprueba rapido,
4. el foco esta muy claro,
5. la complejidad operativa es baja.

### Tramo medio del rango

Aplica cuando:

1. hay mas de un activo relevante,
2. se necesita cierto trabajo de ajuste y seleccion,
3. existen uno o dos canales clave,
4. la aprobacion requiere algo mas de coordinacion.

### Tramo alto del rango

Aplica cuando:

1. la complejidad del cliente es mayor,
2. existe mas trabajo de foco y priorizacion,
3. hay varios activos o canales por alinear,
4. el riesgo comercial de hacerlo mal es mas alto,
5. aun sin salir del alcance base, el servicio exige mas supervision y mas criterio.

## 7. Regla de Complejidad por Cantidad de Productos u Ofertas

Aqui esta una decision importante:

la complejidad del cliente no debe trasladarse automaticamente al alcance del paquete.

Por eso conviene separar a los clientes asi:

### Escenario A. Cliente con una sola oferta principal

Es el escenario ideal para el modelo inicial.

La agencia puede trabajar con alta claridad porque:

1. el mensaje se concentra,
2. la captacion es mas simple,
3. la presencia es mas facil de ordenar,
4. el contenido tiene menos dispersion.

Decision recomendada:

Este tipo de cliente entra perfecto al modelo base sin ajustes mayores.

### Escenario B. Cliente con 2 a 5 productos o servicios

Aqui ya hay complejidad moderada.

El error seria querer vender el paquete como si cubriera de forma profunda las 5 ofertas.

Decision recomendada:

1. se elige una oferta principal,
2. se define una oferta secundaria opcional,
3. el paquete trabaja primero sobre la linea con mayor urgencia o mayor potencial comercial.

Regla:

**hasta 5 ofertas no significa 5 veces el alcance.**

Significa que el diagnostico debe elegir foco.

### Escenario C. Cliente con mas de 5 productos o servicios

Aqui ya no conviene hablar solo de producto individual.

Se debe trabajar por:

1. linea de negocio,
2. familia comercial,
3. oferta ancla,
4. categoria prioritaria.

Decision recomendada:

1. no cubrir todo en el paquete base,
2. vender primero una unidad comercial prioritaria,
3. usar el resultado de esa unidad como caso de expansion posterior.

Regla:

**con mas de 5 productos, la agencia no trabaja todo el inventario; trabaja una prioridad comercial clara.**

### Escenario D. Tiendas o negocios con cientos o miles de productos

Este ya es otro tipo de complejidad.

No conviene meterlo dentro del mismo alcance estandar sin una variante especifica.

Porque ahi cambian por completo:

1. la logica de contenido,
2. la logica de captacion,
3. la estructura de conversion,
4. la necesidad de catalogo,
5. el tipo de operacion comercial,
6. el peso del ecommerce y del feed de productos.

Decision recomendada:

Las tiendas con cientos o miles de productos no deben entrar al portafolio inicial como si fueran un cliente normal de una sola oferta.

Se deben manejar de una de estas formas:

1. trabajando una sola categoria o coleccion prioritaria,
2. trabajando un producto ancla de entrada,
3. o creando despues una variante especifica para catalogos amplios y ecommerce.

## 8. Regla Comercial para Clientes de Mucha Complejidad

Si el cliente tiene demasiadas ofertas, demasiadas categorias o demasiados productos, la respuesta comercial correcta no es ampliar sin limite el paquete.

La respuesta correcta es:

1. acotar el foco,
2. elegir prioridad,
3. trabajar una unidad comercial,
4. probar resultado,
5. expandir despues con nuevo alcance.

## 9. Regla de Precio por Complejidad

La complejidad comercial puede mover el precio, pero no debe moverlo de forma caotica.

Propuesta de lectura:

### Complejidad simple

1. una sola oferta principal,
2. pocos canales,
3. aprobacion rapida,
4. foco claro.

Aqui aplica el precio base del rango.

### Complejidad media

1. varias ofertas,
2. dos o tres canales relevantes,
3. ajustes de mensaje por segmento,
4. mas de una decision comercial interna.

Aqui aplica el tercio medio o alto del rango.

### Complejidad alta

1. multiples lineas de negocio,
2. demasiados productos,
3. muchos aprobadores,
4. estructura comercial mas compleja,
5. mas trabajo de seleccion y foco.

Aqui no conviene solo subir precio.

Conviene redefinir alcance y trabajar por fases o por unidad focal.

## 10. Recomendacion Ejecutiva de Cierre de Sprint

Para mantener rentabilidad y claridad, la agencia deberia operar con estas reglas:

1. cada producto tiene un tiempo minimo y maximo razonable,
2. cada producto tiene un rango sugerido de precio,
3. los clientes complejos no amplian automaticamente el alcance,
4. el foco comercial siempre debe reducirse a una unidad prioritaria,
5. los negocios con catalogos grandes requieren variante posterior o trabajo por categoria.

## 11. Conclusión

Si la agencia quiere ser simple, escalable y rentable, no puede intentar cubrir del mismo modo:

1. un negocio con una sola oferta,
2. una empresa con cinco servicios,
3. una tienda con mil productos.

El sistema debe distinguirlos.

La clave no es venderle menos al cliente complejo. La clave es venderle foco antes de venderle amplitud.

La regla final de este documento es:

**a mayor complejidad del cliente, mayor necesidad de acotar la unidad comercial focal antes de ejecutar.**
