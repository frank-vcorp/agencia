# SPEC ARCH-20260423-09

## Titulo

Producto 01: Lanzamiento Comercial Inicial

## Estado

Borrador de definicion ejecutiva

## Fecha

2026-04-23

## Proposito del Producto

Lanzamiento Comercial Inicial es el producto de arranque para empresas nuevas, marcas por lanzar o proyectos que todavia no cuentan con una base comercial digital clara.

Su funcion no es construir la empresa completa ni desarrollar una identidad premium profunda. Su funcion es dejar al cliente listo para salir al mercado de forma seria, clara y funcional.

## Problema que Resuelve

Este producto resuelve situaciones como:

1. el negocio esta por iniciar y no tiene presencia digital,
2. existe una idea o proyecto pero no una estructura comercial minima,
3. la empresa no tiene una forma clara de presentarse,
4. no hay activos minimos para captar interes o contactos,
5. el cliente necesita salir rapido sin pasar por procesos lentos o demasiado artesanales.

## Tipo de Cliente al que Aplica

Aplica principalmente a:

1. empresas nuevas,
2. proyectos por lanzar,
3. unidades de negocio nuevas dentro de empresas ya existentes,
4. marcas que necesitan una salida al mercado rapida y ordenada,
5. clientes pequeños o medianos que necesitan verse serios sin construir desde el dia uno una estructura compleja.

## No Aplica Bien a

No aplica bien a:

1. marcas ya consolidadas que requieren rebranding profundo,
2. empresas que ya tienen presencia y lo que necesitan es captacion,
3. clientes que esperan desarrollo web complejo,
4. clientes que buscan identidad corporativa premium completa,
5. proyectos sin decision clara o sin oferta minimamente definida.

## Objetivo del Producto

Dejar al cliente con una base comercial minima funcional para presentarse, comunicar su oferta y recibir primeros contactos o prospectos.

## Resultado Esperado

Al terminar este producto, el cliente debe quedar con:

1. una propuesta base entendible,
2. una presencia digital minima y seria,
3. una landing o estructura de entrada funcional,
4. un mecanismo claro de contacto o conversion,
5. activos suficientes para iniciar salida al mercado.

## Promesa Correcta del Producto

No prometemos construir una marca completa.

Prometemos dejar una base comercial lista para salir a mercado con claridad, velocidad y orden.

## Entregables Base

Los entregables iniciales de este producto deberian ser cerrados y consistentes.

### Entregables propuestos

1. diagnostico rapido de arranque,
2. definicion base del mensaje comercial,
3. estructura corta de oferta o propuesta de valor,
4. una landing page funcional o pagina de arranque,
5. configuracion de un canal principal de contacto o conversion,
6. piezas visuales minimas para salida inicial,
7. configuracion basica de presencia digital segun el caso,
8. checklist de salida al mercado.

## Activos Reales del Producto

Para que el producto sea entendible y vendible, debe explicitar los activos concretos que pueden formar parte del paquete.

## Activos Base Incluidos

Los activos base incluidos deben representar la infraestructura minima funcional del lanzamiento.

Propuesta inicial de activos base:

1. landing page o pagina de arranque,
2. configuracion de formulario de contacto o boton a WhatsApp,
3. copy base de presentacion comercial,
4. estructura de propuesta de valor inicial,
5. configuracion basica de identidad visual funcional,
6. una tanda minima de piezas graficas de lanzamiento,
7. texto base de descripcion comercial para perfiles o pagina,
8. checklist final de salida a mercado.

## Activos Opcionales Segun Diagnostico

Los activos opcionales no deben incluirse de forma automatica. Deben activarse solo si el diagnostico los justifica y si estan dentro del alcance pactado.

Propuesta inicial de activos opcionales:

1. alta o configuracion de Google Business Profile,
2. alta o configuracion de pagina de Facebook,
3. alta o configuracion de cuenta de Instagram,
4. alta o configuracion de pagina de LinkedIn,
5. adaptaciones adicionales por canal,
6. piezas extra de lanzamiento,
7. estructura base para una segunda pagina o seccion simple,
8. configuracion minima de un canal adicional de contacto.

## Criterio sobre Activos

No todos los clientes deben recibir todos los activos.

Este producto debe operar por seleccion de activos pertinentes, no por acumulacion indiscriminada.

La seleccion correcta depende de:

1. el tipo de negocio,
2. el canal principal de captacion,
3. el nivel de madurez del cliente,
4. la urgencia de salida,
5. el alcance comercial acordado.

La promesa correcta no es "te damos todo".

La promesa correcta es:

**te dejamos montados los activos minimos correctos para salir a mercado de forma seria y funcional.**

## Incluye

El producto puede incluir, en su version inicial estandar:

1. sesion breve de discovery,
2. organizacion del mensaje principal,
3. copy base para presentacion comercial,
4. estructura de landing o mini sitio,
5. diseño funcional basico,
6. llamada a la accion definida,
7. integracion simple a WhatsApp, formulario o medio de contacto,
8. piezas o adaptaciones visuales minimas para publicar o presentar.

Cuando aplique segun diagnostico y alcance cerrado, tambien puede incluir altas o configuraciones iniciales de perfiles clave como Google Business Profile, Facebook, Instagram o LinkedIn.

## Subcomponente Opcional: Impulso Inicial de Demanda

Cuando el cliente cumple condiciones minimas de claridad, punto de conversion y capacidad de atencion, este producto puede integrar un subcomponente opcional llamado Impulso Inicial de Demanda.

Su funcion es activar una prueba breve y controlada en canales como Google Ads o Meta Ads para validar dos cosas:

1. si el mercado responde al mensaje,
2. si el cliente esta listo para recibir y atender demanda.

Este subcomponente no equivale al producto completo de captacion.

Es una activacion tactica temprana para probar readiness comercial.

### Puede incluir

1. configuracion basica de una campaña inicial,
2. una pieza o variante creativa minima,
3. copy de anuncio base,
4. activacion hacia landing o canal de contacto ya preparado,
5. lectura inicial de respuesta.

### No debe implicar

1. gestion mensual completa de pauta,
2. optimizacion avanzada continua,
3. multiples experimentos simultaneos,
4. promesa de volumen sostenido,
5. sustitucion del producto Funnel de Captacion.

## No Incluye

Para proteger alcance y rentabilidad, este producto no debe incluir por defecto:

1. branding profundo,
2. manual de marca completo,
3. sitio web complejo con multiples secciones,
4. ecommerce,
5. automatizaciones avanzadas,
6. gestion mensual de pauta,
7. estrategia de contenido recurrente,
8. fotografia o video premium extensivo,
9. multiples rondas abiertas de cambios,
10. impresion, fabricacion o entrega fisica directa de materiales.

## Regla sobre Activos Fisicos

Este producto pertenece a una agencia de ejecucion digital.

Por lo tanto:

1. los entregables base son digitales,
2. no se incluyen impresos,
3. no se incluyen materiales tangibles producidos directamente,
4. no se contempla logistica de impresion dentro del alcance normal.

Si el cliente requiere una pieza fisica puntual, como un material promocional pequeno para activar reseñas o reforzar presencia, eso solo podra manejarse como excepcion:

1. fuera del alcance base,
2. con costo separado,
3. mediante aliado o proveedor tercero,
4. sin convertir el producto en un servicio fisico recurrente.

## Que Si Recibe el Cliente

El cliente recibe una base comercial digital real para salir al mercado con orden y funcionalidad.

Eso puede traducirse en:

1. una presencia inicial clara,
2. una pagina o punto digital de entrada,
3. canales digitales listos para operar,
4. piezas base para comunicar su salida,
5. estructura minima para captar interes o contacto,
6. activos digitales montados y utilizables.

## Que No Debe Esperar Recibir

El cliente no debe interpretar este producto como un servicio abierto de construccion total de marca o despliegue integral sin limites.

No debe esperar recibir dentro de este paquete:

1. branding premium completo,
2. sitio web complejo,
3. campaña permanente de marketing,
4. produccion audiovisual de alto nivel,
5. impresos o materiales fisicos incluidos,
6. estrategia ilimitada,
7. revisiones abiertas sin control,
8. todos los canales posibles montados al mismo tiempo.

La promesa del producto es concreta:

**dejar una base comercial digital funcional, no resolver toda la infraestructura futura del negocio dentro de un solo paquete.**

## Prerrequisitos Minimos

Para poder ejecutar este producto, el cliente debe tener al menos:

1. una oferta o servicio minimamente definible,
2. capacidad real de atender prospectos o contactos,
3. una persona que pueda aprobar con rapidez,
4. insumos minimos sobre su negocio,
5. decision real de salir al mercado en corto plazo.

## Flujo de Ejecucion del Producto

El flujo sugerido es:

1. prediagnostico de madurez,
2. confirmacion de que el cliente esta en etapa de lanzamiento,
3. recopilacion de insumos,
4. definicion de mensaje y estructura base,
5. produccion de landing y activos minimos,
6. revision y ajuste controlado,
7. entrega y checklist de salida.

## Criterios de Supervisión Humana

Como es un producto AI-assisted, la supervision humana debe revisar explicitamente:

1. claridad del mensaje,
2. coherencia comercial,
3. nivel minimo de calidad visual,
4. consistencia entre oferta y llamada a la accion,
5. ausencia de salidas genericas o vacias producidas por IA.

## Criterios de Calidad

El producto se considera bien entregado si:

1. el cliente puede explicar su oferta con mayor claridad,
2. existe una pagina funcional lista para compartir,
3. el canal de contacto esta activo,
4. la presencia inicial transmite orden y seriedad,
5. la salida al mercado puede hacerse sin depender de piezas adicionales criticas.

## Riesgos del Producto

Los principales riesgos a controlar son:

1. que el cliente quiera meter branding completo dentro del mismo paquete,
2. que el proyecto no tenga oferta definida,
3. que se convierta en desarrollo web disfrazado,
4. que se exijan demasiadas iteraciones creativas,
5. que se prometan resultados de captacion sin haber vendido aun ese siguiente producto.

## Nivel en la Escalera del Cliente

Este producto corresponde al nivel mas temprano de la escalera.

Resuelve la necesidad de salir correctamente al mercado.

## Siguiente Escalon Natural

Si el producto funciona y el cliente ya tiene base operativa minima, los siguientes escalones naturales pueden ser:

1. Presencia que Vende, si la salida inicial requiere mejorar estructura y conversion,
2. Funnel de Captacion, si el cliente ya esta listo para generar demanda,
3. Motor de Contenido Funcional, si necesita sostener comunicacion y confianza.

## Criterio de Ready for Upsell

El cliente puede subir al siguiente nivel si:

1. ya tiene una base funcional publicada,
2. entiende y aprueba su mensaje comercial,
3. puede atender contactos,
4. existe una siguiente necesidad clara de conversion, captacion o contenido,
5. no quedan huecos criticos de base que impidan avanzar.

Si se activo Impulso Inicial de Demanda, el readiness tambien puede evaluarse con base en:

1. capacidad real de respuesta del cliente,
2. calidad de los contactos recibidos,
3. comportamiento del punto de conversion,
4. viabilidad de escalar captacion formal.

## Valor Estratégico para la Agencia

Este producto sirve como puerta de entrada de alta demanda para clientes pequeños y medianos.

Su valor estrategico esta en que:

1. resuelve una urgencia clara,
2. es facil de explicar comercialmente,
3. puede estandarizarse bien,
4. abre la puerta a productos de mayor ticket,
5. ayuda a construir confianza rapida.

## Riesgo Económico para la Agencia

Este producto solo sera rentable si se mantiene cerrado.

Pierde rentabilidad cuando:

1. se convierte en consultoria abierta,
2. se vuelve branding premium escondido,
3. se amplian entregables sin control,
4. se aceptan demasiadas revisiones,
5. se vende a clientes que todavia no tienen claridad minima.

## Decision Provisional

Lanzamiento Comercial Inicial se mantiene como producto candidato valido dentro del portafolio inicial de la agencia.

## Pendientes para Cierre Final

Antes de declararlo cerrado como paquete comercial definitivo, aun falta definir:

1. nombre comercial final,
2. entregables exactos y numero maximo,
3. formato de revision y limites de cambios,
4. tiempo de entrega,
5. rango de precio,
6. assets minimos requeridos por parte del cliente.
